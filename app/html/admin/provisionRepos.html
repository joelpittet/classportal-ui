Github Repo Generation

WARNING: Repo generation is a time-consuming process that takes approximately 5 seconds per repository. Since API calls can occasionally fail, a script to fix minor errors can also be run based on the need.

Github repositories CANNOT be deleted once they are created.

If students have not logged into Github Enterprise, then they will not visible in Github Enterprise. This will lead their respective repo creation to fail. The repair script can be run periodically to re-add repository teams members after they have re-logged in.

Error Manager

WARNING: This will PERMANENTLY disassociate all Github linkages for the Teams created under the current Deliverable and Github's Repositories.

This operation should only if it is essential that all Github Repos must be re-created, either due to an error in business logic or a critical error during the Github Repo creation process.

STEPS TO RECOVERY:

(1.) A Github Organization owner must manually delete all of the Github Repos associated with this Deliverable.

(2.) The Team object links to the Github Repos can then be erased by clicking on "Erase Github Repo".

(3.) Click on "Provision Repos".
