Github Repo Generation

WARNING: Repo generation is a time-consuming process that takes approximately 5 seconds per repository. Since API calls can occasionally fail, a script to fix minor errors can also be run based on the need.

Github repositories CANNOT be deleted once they are created without Github Organization permissions.

If students have not logged into Github Enterprise, then they will not be visible in Github Enterprise. This will result in the creation of their respective repo to fail. The repair script can be run periodically to re-add team members to a repository after they have logged in to Github.

Error Manager

WARNING: This will PERMANENTLY disassociate all Github linkages from Teams created under the current Deliverable and Github's Repositories.

DO NOT perform this operation unless it is essential that all Github Repos must be re-created due to a critical error.

STEPS TO RECOVERY:

(1.) A Github Organization owner must manually delete all of the Github Repos associated with this Deliverable.

(2.) The Team object links to the Github Repos can then be erased by clicking on "Erase Github Repo".

(3.) Click on "Provision Repos".
