Destroy Manager

WARNING: Destroying a Docker container can have detrimental effects if it is currently being used in a course.

If you are re-building the container, ensure that all Dockerfile dependencies are still valid, by building the container in your test environment, before deleting the active container. Any unknown deprecated dependencies may result in a serious disruption to your course if you cannot successfully re-build your container.

  1. Dockerfile must be included in the base-level of Git Repository.
  2. Dockerfile must run from start-to-finish without error.
  3. Dockerfile repository must be publicly accessible with read permissions or a token must be provided to access the repository.
